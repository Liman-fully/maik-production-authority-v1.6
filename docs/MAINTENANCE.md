## 🧹 垃圾清理

### 清理范围

| 类型 | 保留策略 | 清理频率 |
|------|---------|---------|
| 临时文件 | 不保留 | 每日 |
| 日志文件 | 30 天 | 每日 |
| Docker 悬空镜像 | 不保留 | 每日 |
| Git 垃圾 | 不保留 | 每日 |
| 旧备份 | 30 天 | 每日 |
| node_modules | 不保留 | 每周 |

### 清理脚本

```bash
# 每日自动执行
./scripts/cleanup.sh

# 手动执行
npm run cleanup
```

## 📝 日志治理

### 日志分类

| 类型 | 位置 | 保留策略 |
|------|------|---------|
| 应用日志 | `logs/app.log` | 30 天 |
| 错误日志 | `logs/error.log` | 90 天 |
| 访问日志 | `logs/access.log` | 30 天 |
| 部署日志 | `logs/deploy.log` | 90 天 |

### 日志轮转

```bash
# 每日自动执行
./scripts/rotate-logs.sh

# 日志大小限制：100MB/文件
# 保留文件数：30 个
```

### 日志查看

```bash
# 查看最新日志
tail -f logs/app.log

# 查看错误日志
cat logs/error.log

# 搜索日志
grep "关键词" logs/*.log
```

## 💾 备份机制

### 备份范围

| 内容 | 频率 | 保留时间 | 位置 |
|------|------|---------|------|
| 数据库 | 每日 | 30 天 | `backups/db/` |
| 配置文件 | 每日 | 30 天 | `backups/config/` |
| 代码 | 实时 | 永久 | GitHub |
| 文档 | 实时 | 永久 | GitHub |

### 备份脚本

```bash
# 每日自动执行（06:00）
./scripts/backup-db.sh

# 手动备份
npm run backup
```

### 恢复流程

```bash
# 1. 停止服务
docker-compose down

# 2. 恢复数据库
./scripts/restore-db.sh [备份文件]

# 3. 启动服务
docker-compose up -d

# 4. 验证恢复
./scripts/health-check.sh
```

## 📦 依赖管理

| 类型 | 策略 | 频率 |
|------|------|------|
| 补丁版本 | 自动更新 | 每周 |

### 更新流程

```bash
npm update

npm test

git add package.json package-lock.json
git commit -m "chore: 更新依赖"

```

## 🚨 故障处理

### 故障分级

| 级别 | 说明 | 响应时间 | 处理人 |
|------|------|---------|--------|
| P0 | 服务不可用 | 5 分钟 | 全体 |
| P1 | 核心功能故障 | 30 分钟 | 都统 |
| P2 | 非核心功能故障 | 2 小时 | 协调者 |
| P3 | 优化建议 | 1 周 | 全体 |

### 故障处理流程

```
1. 发现故障 → 立即通知@全体
2. 评估影响 → 确定故障级别
3. 紧急修复 → 优先恢复服务
4. 根因分析 → 防止再次发生
5. 经验总结 → 记录到文档
```
