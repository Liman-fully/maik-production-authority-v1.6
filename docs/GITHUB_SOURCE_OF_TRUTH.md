## 🎯 三大原则

### 1️⃣ 唯一共享空间
- **所有代码、文档、配置**必须存储在 GitHub 仓库
- **禁止**使用任何第三方存储作为共享渠道
- **禁止**通过非官方途径（微信、QQ、邮件等）传递项目文件

### 2️⃣ 唯一可信来源
- **所有信息获取**：本地工作区 ← → GitHub 云端
- **本地内容不可信**，GitHub 内容才可信
- **跨设备协作**时，以 GitHub 为唯一中转站

### 3️⃣ 实时同步
- **代码**：实时同步，每次提交必须 push
- **文档**：实时同步，更新后必须 push
- **配置**：实时同步，修改后必须 push

## 📋 执行规则（强制）

### ✅ 必须做的事

| 场景 | 操作 | 命令 |
|------|------|------|
| **开始工作前** | 从 GitHub 获取最新代码 | `git pull origin master` |
| **完成任务后** | 立即 push 到 GitHub | `git push origin master` |
| **离开前** | 确保所有代码已 push | `git status` 检查 |
| **新成员加入** | 从 GitHub 克隆完整项目 | `git clone <URL>` |
| **跨设备工作** | 先 push 再 pull | `git push` → `git pull` |
| **文档更新** | 与代码同步 push | `git add && commit && push` |
| **状态确认** | 查看 GitHub 提交历史 | GitHub → Commits |

### ❌ 禁止做的事

| 行为 | 风险 | 处罚 |
|------|------|------|
| 本地代码不 push | 丢失风险 | 通报批评 |
| 通过非 GitHub 渠道传递文件 | 版本混乱 | 严重警告 |
| 依赖本地记忆不查 GitHub | 信息错误 | 事故报告 |
| 假设其他 Agent 知道本地内容 | 协作失误 | 绩效扣减 |
| 使用第三方云盘同步项目 | 安全风险 | 立即禁止 |

## 🔧 验证方法

### 每日验证（每次工作前）

```bash
# 1. 验证远程仓库连接
git remote -v
# 应显示：origin https://github.com/Liman-fully/huntlink.git

# 2. 获取最新代码
git pull origin master

# 3. 查看 GitHub 提交历史
git log --oneline -10

# 4. 验证本地与远程同步
git status
# 应显示：nothing to commit, working tree clean
```

## 🚨 违规处理

### 违规分级

| 级别 | 行为 | 处罚 |
|------|------|------|
| **P0** | 故意不 push 代码导致团队工作受阻 | 事故报告 + 绩效扣减 |
| **P1** | 通过非 GitHub 渠道传递项目文件 | 严重警告 + 重新培训 |
| **P2** | 忘记 push 但及时纠正 | 提醒 + 记录 |
| **P3** | 首次违规且无影响 | 口头提醒 |

### 事故报告流程

```
1. 发现违规 → 立即记录
2. 评估影响 → 确定级别
3. 编写报告 → docs/INCIDENT_REPORT_XXX.md
4. 团队通报 → @全体成员
5. 整改跟踪 → 确保不再发生
```

## 📞 常见问题

### Q1: 如果 GitHub 访问失败怎么办？

**A**: 
1. 检查网络连接
2. 检查 GitHub 状态：https://www.githubstatus.com/
3. 如无法访问，等待恢复后再工作
4. **禁止**使用其他方式同步

### Q2: 本地修改还没完成，需要 push 吗？

**A**:
- 小修改：完成后立即 push
- 大功能：创建 feature 分支，定期 push 到该分支
- **原则**：不要长时间不 push

### Q3: 如何确认其他 Agent 的工作？

**A**:
1. 查看 GitHub 提交历史
2. 查看 `.task-board.md`
3. 查看 `docs/TODAY_PROGRESS.md`
4. **不要**假设，**要**查证

### Q4: 新 Agent 如何加入？

**A**:
```bash
# 1. 克隆项目
git clone https://github.com/Liman-fully/huntlink.git

# 2. 阅读文档
cd huntlink
cat README.md
cat docs/QUICK_START.md

# 3. 登记身份
编辑 .task-board.md 添加自己的信息
```
