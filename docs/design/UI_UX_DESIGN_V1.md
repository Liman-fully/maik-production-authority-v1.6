# 猎脉 HuntLink UI/UX 设计方案 V1.0

> **设计师**: 左护法（天策府主官）  
> **创建时间**: 2026-03-28  
> **设计原则**: 人性化、高效、专业、温暖

---

## 一、现状问题分析

### 当前问题
1. **Dashboard（首页）**
   - ❌ 信息密度过低，浪费屏幕空间
   - ❌ 快捷入口按钮过大，视觉沉重
   - ❌ 待办事项缺少优先级标识
   - ❌ 缺少数据可视化（图表、趋势）

2. **TalentMarket（人才广场）**
   - ❌ 搜索框太小，不符合核心功能定位
   - ❌ 筛选条件隐藏过深，使用不便
   - ❌ 简历卡片信息过载，重点不突出
   - ❌ 缺少批量操作入口

3. **ResumeLibrary（简历库）**
   - ❌ 左侧文件夹占用空间过大（25%）
   - ❌ 邮箱导入功能入口不明显
   - ❌ 标签页设计不合理（用 Tag 当 Tab）
   - ❌ 批量操作按钮位置不当

---

## 二、设计理念

### 核心原则
```
1. 效率优先 - 核心功能 3 次点击内到达
2. 信息分层 - P0 信息突出，P1 信息可折叠，P2 信息隐藏
3. 视觉舒适 - 留白适度，色彩统一，对比度合理
4. 操作流畅 - 减少页面跳转，支持快捷键，批量操作
```

### 色彩方案
```
主色调:
- 品牌蓝：#166BFF (专业、信任)
- 成功绿：#00A870 (积极、完成)
- 警告橙：#FF8800 (注意、待处理)
- 危险红：#EB3434 (紧急、错误)

中性色:
- 标题：#1D2129 (深灰)
- 正文：#4E5969 (中灰)
- 次要：#86909C (浅灰)
- 边框：#E5E6EB (极浅灰)
- 背景：#F7F8FA (白色背景)
```

### 字体规范
```
标题:
- H1: 24px / 600 (页面标题)
- H2: 20px / 600 (模块标题)
- H3: 16px / 600 (卡片标题)

正文:
- 大号：16px / 400 (重要信息)
- 常规：14px / 400 (普通文本)
- 小号：12px / 400 (辅助信息)

行高:
- 标题：1.4
- 正文：1.6
- 紧凑：1.3
```

**设计目标**：数据驱动、高效导航、一目了然

**布局**：

```
│  欢迎区（简洁）                                     │
│  "早上好，左护法 · 今天有 12 份新简历" [查看]        │
├──────────────┬──────────────────────────────────────┤
│  核心指标    │  快捷操作                            │
│  (4 个卡片)   │  (6 个按钮，2 行 3 列)                │
│              │                                      │
│  [新增 128]  │  [发布职位] [搜索人才] [导入简历]    │
│  [总量 3842] │  [批量面试] [导出报表] [邮箱设置]    │
│  [待处理 56] │                                      │
│  [活跃 8]    │  待办事项（时间线样式）               │
│              │  ┌──────────────────────────────┐   │
│              │  │ 14:00 面试 张** 前端工程师    │   │
│              │  │ 15:30 电话 李** 产品意向确认  │   │
│              │  │ 明日  跟进 王** offer 谈判    │   │
│              │  └──────────────────────────────┘   │
├──────────────┴──────────────────────────────────────┤
│  数据趋势（图表）                                   │
│  [本周简历趋势] [职位效果对比] [渠道来源分析]       │

```

**关键改进**：
- ✅ 核心指标从 4 个精简到 3 个（去掉不重要的）
- ✅ 快捷操作从 4 个增加到 6 个（2 行 3 列）
- ✅ 待办事项改用时间线样式，更直观
- ✅ 新增数据趋势图表，辅助决策

**代码示例**：

### 3.3 ResumeLibrary（简历库）- 重新设计

**设计目标**：空间优化、操作便捷、状态清晰

**布局**：
```
┌─────────────────────────────────────────────────────┐
│  顶部工具栏                                         │
│  [搜索简历...]  [上传] [批量操作] [邮箱设置]        │
├─────────┬───────────────────────────────────────────┤
│  文件夹  │  标签页                                  │
│  (折叠)  │  [全部 156] [待处理 12] [已解析 140]     │
│          │                                          │
│  ▶ 全部  │  简历列表                                │
│  ▶ 前端  │  ┌────────────────────────────────┐     │
│  ▶ 后端  │  │ [简历卡片 1]                   │     │
│  ▶ 产品  │  │ [简历卡片 2]                   │     │
│  ▶ 设计  │  │ ...                            │     │
│          │  └────────────────────────────────┘     │
│  ──────  │                                          │
│  📧 邮箱  │  分页器                                 │
│  已连接 2  │  [<] 1 2 3 4 5 [>]  每页 20 条          │
│  今日 15  │                                          │
└─────────┴───────────────────────────────────────────┘
```

**关键改进**：
- ✅ 左侧文件夹可折叠，默认收起（节省空间）
- ✅ 标签页从 Tag 改为标准 Tabs 组件
- ✅ 邮箱状态简化显示（已连接 + 今日导入）
- ✅ 分页器标准化，支持每页数量选择

**代码示例**：
```tsx
// 文件夹组件（可折叠）
<Card 
  title={
    <div style={{ display: 'flex', justifyContent: 'space-between' }}>
      <Text>简历文件夹</Text>
      <Button theme="default" variant="text" icon={<FoldIcon />} />
    </div>
  }
  className="folder-panel"
  style={{ width: collapsed ? 48 : 200 }}
>
  {!collapsed && (
    <>
      <Button theme="primary" variant="dashed" block icon={<PlusIcon />}>
        新建文件夹
      </Button>
      <Tree data={folders} />
    </>
  )}
</Card>
```

## 四、组件设计规范

### 4.1 简历卡片（核心组件）

**设计原则**：信息分层、重点突出、操作便捷

**布局**：
```
┌─────────────────────────────────────────────────────┐
│  ☑  [头像] 张**  高级前端工程师  @字节跳动          │
│      5 年 · 本科 · 清华大学 · 北京 · 40-60K        │
├─────────────────────────────────────────────────────┤
│  技能：React TypeScript Node.js Vue                │
│  状态：2 小时前活跃  匹配度 95%  个人评分 88        │
├─────────────────────────────────────────────────────┤
│  [查看简历] [下载] [发送面试] [加入人才库] [更多▼]  │
└─────────────────────────────────────────────────────┘
```

**交互**：
- 悬停：卡片轻微上浮 + 阴影加深
- 点击：整张卡片可点击（除操作按钮外）
- 勾选：批量操作模式显示勾选框

**代码示例**：
```tsx
<Card 
  className="resume-card"
  hoverable
  onClick={() => onViewResume(data)}
>
  <div className="resume-header">
    <Checkbox 
      checked={selected} 
      onChange={onSelect}
      onClick={(e) => e.stopPropagation()}  // 防止触发卡片点击
    />
    <Avatar src={data.avatar} size={48} />
    <div className="resume-info">
      <Text strong size="large">{data.name}</Text>
      <Text secondary>{data.currentTitle} · {data.currentCompany}</Text>
      <Text secondary>{data.experience} · {data.education}</Text>
    </div>
    <div className="resume-scores">
      <Tag theme="primary" variant="light">匹配度 {data.matchScore}%</Tag>
      <Tag theme="success" variant="light">评分 {data.personalScore}</Tag>
    </div>
  </div>
  
  <div className="resume-skills">
    <Text secondary>技能：</Text>
    {data.skills.map(skill => (
      <Tag key={skill} variant="outline">{skill}</Tag>
    ))}
  </div>
  
  <div className="resume-actions">
    <Button theme="primary">查看简历</Button>
    <Button variant="outline">下载</Button>
    <Button variant="outline">发送面试</Button>
    <Button variant="outline">加入人才库</Button>
  </div>
</Card>
```
