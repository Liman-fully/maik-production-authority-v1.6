#### 经验 1: 低配服务器（2G RAM）下的 Node.js 内存治理
**场景**: 2G 内存限制下进行 NestJS 后端开发。
**做法**: 
1. 模块化按需加载，避免在 AppModule 中导入过多无用 Service。
2. 数据聚合（Statistics）尽量使用 SQL 原生计算，避免在 Node.js 内存中处理大型 Candidate 数组。
3. 严格控制 Redis 缓存淘汰策略，防止缓存撑爆内存。
**效果**: 系统在开发阶段保持稳定，未出现 OOM（内存溢出）导致的服务崩溃。

#### 经验 2: 前端图表性能优化（Recharts）
**场景**: 升级 Dashboard 可视化图表。
**做法**: 
1. 选用轻量级的 Recharts 库，并配合 ResponsiveContainer 实现响应式布局。
2. 数据采样：在后端完成数据聚合（Funnel/Trend），前端只负责绘制，不负责逻辑计算。
**效果**: 页面渲染速度极快，CPU 占用率低。

#### 经验 3: “GitHub 唯一真理”在受限环境下的实践
**场景**: 本地 Git Push 权限受限或环境不稳时。
**做法**: 
1. 即使 Push 失败，也要第一时间更新 `docs/TODAY_PROGRESS.md` 和 `docs/experiences/`。
2. 每一个 Commit 必须包含任务 ID（如 feat(DEV-V3.2): xxx），确保追溯链条不断裂。
**效果**: 成功消除了与主公（用户）之间的信息差，确保了“军机不外泄且不错漏”。
