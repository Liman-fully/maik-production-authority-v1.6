# 猎脉项目 - 开发者快速入门指南

## 🎯 项目概述

猎脉(HuntLink)是一个基于NestJS + Next.js的全栈招聘社交平台，专为极低资源环境优化，集成腾讯云COS存储和PostgreSQL全文搜索。

**项目地址**: https://github.com/Liman-fully/huntlink-new  
**部署环境**: 腾讯轻量云 150.158.51.199  
**技术栈**: NestJS + Next.js + PostgreSQL + Redis + 腾讯云COS

---

## 📋 环境要求

### 必需
- **Node.js**: 20.x 或更高版本（建议使用管理版本）
- **npm**: 随Node.js一起安装

### 推荐
- **Docker**: 用于运行PostgreSQL和Redis
- **Docker Compose**: 一键启动所有服务

## 🚀 快速开始（5分钟）

### 方式一：自动配置（推荐）

```bash
# 1. 克隆项目（如果尚未克隆）
git clone https://github.com/Liman-fully/huntlink-new.git
cd huntlink-new

# 2. 运行环境配置向导
bash scripts/setup-env.sh

# 3. 运行环境检查
bash scripts/quick-env-check.sh

# 4. 启动所有服务
docker-compose up -d

# 5. 查看日志
docker-compose logs -f
```

### 方式二：手动配置

```bash
# 1. 克隆项目
git clone https://github.com/Liman-fully/huntlink-new.git
cd huntlink-new

# 2. 安装Node.js（如需要）
# 使用 install_binary 工具安装管理版本

# 3. 安装依赖
cd backend && npm install
cd ../frontend-web && npm install

# 4. 配置环境变量
cp backend/.env.example backend/.env
cp frontend-web/.env.example frontend-web/.env

# 5. 编辑.env文件，配置数据库和COS密钥
# 使用你喜欢的编辑器: vim/nano/code

# 6. 启动服务
# 选项A: Docker方式（推荐）
cd ..
docker-compose up -d

# 选项B: 手动方式
docker-compose up -d postgres redis  # 启动数据库
cd backend && npm run start:dev       # 启动后端
cd ../frontend-web && npm run dev     # 启动前端
```

## 🔧 详细配置指南

### 1. 环境变量配置

#### 后端配置 (backend/.env)

```bash
# 数据库配置
DB_HOST=localhost           # Docker环境下改为 postgres
DB_PORT=5432
DB_USERNAME=huntlink
DB_PASSWORD=your_password   # 修改为实际密码
DB_DATABASE=huntlink

# 腾讯云COS配置（必需）
COS_SECRET_ID=your_secret_id      # 从腾讯云控制台获取
COS_SECRET_KEY=your_secret_key    # 从腾讯云控制台获取
COS_BUCKET=huntlink-1306109984
COS_REGION=ap-guangzhou
COS_DOMAIN=huntlink-1306109984.cos.ap-guangzhou.myqcloud.com

# Redis配置
REDIS_HOST=localhost        # Docker环境下改为 redis
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# JWT配置
JWT_SECRET=your_random_key  # 使用openssl或node生成
JWT_EXPIRATION=7d

# 服务器配置
PORT=3000
NODE_ENV=development
```

#### 前端配置 (frontend-web/.env)

```bash
# API基础地址
VITE_API_BASE_URL=http://localhost:3000/api  # 生产环境改为实际地址

# 应用配置
VITE_APP_NAME=猎脉 HuntLink
VITE_APP_VERSION=1.0.0
```

### 2. 生成JWT密钥

```bash
# 方式一: 使用openssl
openssl rand -base64 32

# 方式二: 使用Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 3. 腾讯云COS配置

1. 登录腾讯云控制台
2. 进入对象存储 COS 服务
3. 创建存储桶（或使用现有存储桶）
4. 获取SecretId和SecretKey
5. 配置CORS规则（允许前端直传）

**CORS配置示例**:
```xml
<AllowedOrigin>*</AllowedOrigin>
<AllowedMethod>GET</AllowedMethod>
<AllowedMethod>PUT</AllowedMethod>
<AllowedMethod>POST</AllowedMethod>
<AllowedHeader>*</AllowedHeader>
```

## 🔍 故障排查

### 常见问题

#### 1. Node.js版本不匹配
```bash
# 使用管理版本而非系统版本
/Users/liman/.workbuddy/binaries/node/versions/22.12.0/bin/node
```

#### 2. 数据库连接失败
- 检查PostgreSQL是否运行：`docker ps | grep postgres`
- 检查端口是否占用：`lsof -i :5432`
- 验证配置：检查backend/.env中的数据库配置

#### 3. COS上传失败
- 检查COS密钥是否正确
- 检查存储桶是否存在
- 检查CORS配置
- 查看日志：`docker-compose logs backend`

#### 4. 前端无法连接后端
- 检查后端是否运行：`curl http://localhost:3000/api/health`
- 检查API地址配置：frontend-web/.env
- 检查CORS配置

### 日志查看
```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f postgres
docker-compose logs -f redis
```
