## 👤 给新 Agent 的话
你现在接手的是由 Nova (Liman 的 CTO/CPO) 亲自调优并审计过的生产级工程。本项目拒绝任何形式的“架构降级”或“逻辑退化”。

## 🚨 必须遵守的准则
- **去歧视算法**: 判定职级采用 薪资/公司/职级 优势优先，物理隐藏年龄。
- **隐私掩码**: 全站职场名片默认脱敏，非好友不得查看联系方式。
- **2G RAM 锁死**: 所有的开发必须在 2G 内存限制下通过。
- **更新收益分发**: 谁更新简历，谁获全量收益。

## 🛠 极端环境部署规约 (The 2G-RAM Constraint)
- **禁止在 Server 执行 \`npm install\` 或 \`npm run build\`**: 本地服务器环境受限，且内存不足以支撑高性能编译。
- **构建分发模式 (Build-Artifact-Only)**: 前端开发 Agent 必须在本地完成 \`build\`，并将 \`dist/\` 目录作为一个整体资产推送到服务器或 GitHub 的 \`frontend-dist/\` 目录。
- **环境嗅探**: 在执行任何部署脚本前，Agent 必须首先检查 \`node --version\`。若环境缺失必备编译器，应立即切换至“物理资产搬运”模式，而非报错停止。

## 📍 故障排除流程
1. 若 UI 不刷新 -> 物理检查 \`frontend-dist/\` 下的 \`index.html\` 指纹。
2. 若 Backend 不响应 -> 检查 3000 端口死锁，并使用 \`fuser -k\` 强制清理。
